HTML => структура(разметка)
  не должно быть информации, которая влияет на отображение

  Чем плохая табличная верстка:
    - нельзя изменить без вмешательста в html, т.к уже задана структура (html влияет на представление)

  Хорошая верстка разделает представление и структуру, с помощу css мы начинаем "раскрашивать"
CSS => оформление(представление)


tags- элементы на странице


block <div>element</div>div>
inline <span>element</span>
отличие
  блочный элемент по умолчанию занимает всю отдаваемуему ему ширину.
  линейный элемент занимает только необходимое ему место, как пример буква
  линейным элементам нельзя назначить ширину и высоту

      наглядный пример
      <div>element div</div> A
      <span>element span</span> B

с помощу display мы можем назначать линейным элементам свойства блочного

стили по умолчанию
  это стиль, который назначает браузер. Для того что б этого избежать используют reset css.
    * означает применить ко всему
недостатки
  когда нужно поддерживать лог или что-то еще, где есть дефолтные стили, отресторить стили сложно

10px - 10 точек монитора
10% - от чего будем считать проценты (обычно считаются от 1-го родителя)
10em - высота строки (можно сбрасывать размер шрифта font-size 1 em)
10pc - кегельная шпация

Почему используем class, а не id
id - может быть только 1-н на старницу
  - большой вес(чем больше подклассов имеет элемент, тем больший вес он будет иметь)
